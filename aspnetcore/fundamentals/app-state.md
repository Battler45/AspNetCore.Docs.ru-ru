## <a name="session-state"></a><span data-ttu-id="c03db-145">Состояние сеанса</span><span class="sxs-lookup"><span data-stu-id="c03db-145">Session state</span></span>

<span data-ttu-id="c03db-146">Состояние сеанса — это сценарий ASP.NET Core для хранения пользовательских данных, когда пользователь находится в веб-приложении.</span><span class="sxs-lookup"><span data-stu-id="c03db-146">Session state is an ASP.NET Core scenario for storage of user data while the user browses a web app.</span></span> <span data-ttu-id="c03db-147">Состояние сеанса использует хранилище, обслуживаемое приложением, для сохранения данных между запросами от клиента.</span><span class="sxs-lookup"><span data-stu-id="c03db-147">Session state uses a store maintained by the app to persist data across requests from a client.</span></span> <span data-ttu-id="c03db-148">Данные сеанса поддерживаются кэшем и считаются временными, &mdash; сайт должен работать и без данных сеанса.</span><span class="sxs-lookup"><span data-stu-id="c03db-148">The session data is backed by a cache and considered ephemeral data&mdash;the site should continue to function without the session data.</span></span> <span data-ttu-id="c03db-149">Критически важные данные приложения должны храниться в пользовательской базе данных и кэшироваться в сеансе только в качестве оптимизации производительности.</span><span class="sxs-lookup"><span data-stu-id="c03db-149">Critical application data should be stored in the user database and cached in session only as a performance optimization.</span></span>
